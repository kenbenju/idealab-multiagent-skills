# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Active  |

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a vulnerability in this repository — including prompt injection vectors, data leakage risks in skill templates, or insecure integrations — please report it responsibly:

1. **Email**: security@idealabpartners.com
2. **Subject line**: `[SECURITY] IDEALAB Skills — <short description>`
3. **Include**:
   - Description of the issue
   - Affected skill(s) or file(s)
   - Steps to reproduce
   - Potential impact
   - (Optional) Suggested fix

## Response Timeline

- **Acknowledgement**: Within 48 business hours
- **Initial assessment**: Within 5 business days
- **Fix or mitigation**: Within 30 days for critical issues

## Scope

Within scope:
- Prompt injection vulnerabilities in skill instructions
- Insecure data handling patterns in CRM or Digital Twin skills
- Privacy risks in contact-tracking or persona-modeling workflows

Out of scope:
- Third-party integrations (Zapier, Make, n8n) — report to those vendors
- General AI model behavior from the underlying LLM provider

## Disclosure Policy

IDEALAB PARTNERS follows a **coordinated disclosure** model. We will publicly credit researchers who responsibly report vulnerabilities (unless anonymity is requested).
